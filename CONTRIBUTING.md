# Contributing to Topology-Based Pore Matching for Zeolite Screening

[English](#english) | [中文](#中文)

---

## English

Thank you for your interest in contributing to this project! This document provides guidelines for contributing.

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Topology-Based-Pore-Matching-for-Rapid-Screening-of-Candidate-Zeolites.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes: `git commit -m "Description of your changes"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Submit a Pull Request

### Code Style

- Follow PEP 8 guidelines for Python code
- Use meaningful variable and function names
- Add docstrings to all functions and classes
- Keep functions focused and modular
- Comment complex algorithms

### Testing

- Write unit tests for new features
- Ensure all tests pass before submitting PR
- Include test cases for edge cases
- Maintain or improve code coverage

### Documentation

- Update README.md if you add new features
- Document all public APIs
- Include usage examples
- Keep CHANGELOG.md updated

### Commit Messages

- Use clear, descriptive commit messages
- Start with a verb in present tense (Add, Fix, Update, etc.)
- Reference issue numbers when applicable

### Pull Request Process

1. Ensure your PR description clearly describes the problem and solution
2. Include relevant issue numbers
3. Update documentation as needed
4. Wait for code review and address feedback

### Reporting Issues

- Use issue templates if available
- Provide detailed description of the problem
- Include steps to reproduce
- Specify environment details (OS, Python version, etc.)
- Attach relevant error messages or logs

### Code Review

All submissions require review. We use GitHub pull requests for this purpose.

## 中文

感谢您对本项目的关注！本文档提供了贡献指南。

### 开始贡献

1. Fork 本仓库
2. 克隆您的 fork: `git clone https://github.com/YOUR_USERNAME/Topology-Based-Pore-Matching-for-Rapid-Screening-of-Candidate-Zeolites.git`
3. 创建新分支: `git checkout -b feature/your-feature-name`
4. 进行修改
5. 彻底测试您的更改
6. 提交更改: `git commit -m "修改描述"`
7. 推送到您的 fork: `git push origin feature/your-feature-name`
8. 提交 Pull Request

### 代码规范

- 遵循 PEP 8 Python 代码规范
- 使用有意义的变量和函数名
- 为所有函数和类添加文档字符串
- 保持函数专注和模块化
- 为复杂算法添加注释

### 测试

- 为新功能编写单元测试
- 提交 PR 前确保所有测试通过
- 包含边缘案例的测试用例
- 维持或提高代码覆盖率

### 文档

- 添加新功能时更新 README.md
- 记录所有公共 API
- 包含使用示例
- 保持 CHANGELOG.md 更新

### 提交信息

- 使用清晰、描述性的提交信息
- 以现在时态动词开头（添加、修复、更新等）
- 在适用时引用 issue 编号

### Pull Request 流程

1. 确保您的 PR 描述清楚地说明问题和解决方案
2. 包含相关的 issue 编号
3. 根据需要更新文档
4. 等待代码审查并处理反馈

### 报告问题

- 如有问题模板，请使用
- 提供问题的详细描述
- 包含重现步骤
- 指定环境详细信息（操作系统、Python 版本等）
- 附上相关的错误消息或日志

### 代码审查

所有提交都需要审查。我们使用 GitHub pull request 进行此操作。
